Synclair's "intelligence" is not a model it ships ‚Äî it's **packaged know-how** your
existing agent loads on demand. It comes in two shapes.

## Skills ‚Äî the packaged know-how

A **skill** is a plain-markdown guide at `.claude/skills/<name>/SKILL.md`, each with a
one-line description of *when* it applies. That's the whole format ‚Äî no framework.

## Agents (diggers) ‚Äî narrow retrieval workers

A **digger** is a sub-agent with its own context window. It reads something heavy ‚Äî a
40-page PRD, the host codebase, a Figma file ‚Äî distills it, and returns one tight
answer. The expensive read happens in a throwaway context; only the answer comes
back. Narrow diggers return tighter answers than one do-everything agent.

Examples that ship: `codebase-surveyor`, `system-mapper`, `knowledge-harvester`,
`token-archaeologist`, `component-cataloger` (the existing-app intake crew),
`prd-retriever`, `figma-frame-reader`, and reviewers like `doc-quality-reviewer`.

## How capabilities are organized

Every skill and agent carries two classifiers, surfaced on `/synclair/ai-setup`:

- **`category`** ‚Äî `build` ¬∑ `knowledge` ¬∑ `intake` ¬∑ `foundation` ¬∑ `tooling` ‚Äî groups them.
- **`layer`** ‚Äî `foundation` (ships with Synclair, syncs from upstream) vs. `project`
  (this repo's own, never syncs) ‚Äî drives the "Origin" badge.

## The gate that keeps the set solid

The set doesn't stay clean by a size limit ‚Äî it stays clean by a **gate at creation**.
Before adding a new skill or agent, you have to prove no existing capability covers it
(extend or distill into it instead), pick skill-vs-agent by whether it needs its own
context window, and declare its layer + category. That gate lives in the `synclair`
skill.

## The flywheel ‚Äî capabilities get smarter

When a digger surfaces something durable, it **writes the durable part back** into the
relevant skill. So the first read of an area is expensive, and every read after is a
cheap one-line load. Over time the distilled brain converges on what a builder
routinely needs.
