The Synclair hub is a single app mounted at **`/synclair`**. Every section is a
*rendering of a file in your repo* — browse the page, or read the source; they're the
same facts. Here's the map.

<Note>
Every section below is one source, two renderings: a human page under `/synclair/*`
and a machine-readable form for agents. Editing the underlying file updates both.
</Note>

## Foundation

<CardGroup cols={2}>
  <Card title="Foundations" icon="palette">
    **`/synclair/foundations`** — the design tokens: color, type, spacing as one
    semantic vocabulary. This is what the lint guardrails point back to.
  </Card>
  <Card title="Components · Blocks · Templates" icon="shapes">
    **`/synclair/components`**, `/blocks`, `/templates` — the component catalog, one
    gallery per tier, each with card previews and a per-item doc page (usage, API,
    UX notes).
  </Card>
</CardGroup>

## Knowledge

<CardGroup cols={2}>
  <Card title="Knowledge" icon="book">
    **`/synclair/knowledge`** — the source-of-truth links (PRDs, Figma, decks) plus
    the distilled summaries and briefs generated per audience. Sources stay canonical
    in Drive/Figma; the repo holds the digests.
  </Card>
  <Card title="References" icon="bookmark">
    **`/synclair/references`** — the project's growing library of prior art,
    findings, and recommendations. Agents append to it as they research.
  </Card>
</CardGroup>

## Visibility into the whole app

<CardGroup cols={2}>
  <Card title="System map" icon="sitemap">
    **`/synclair/system`** — what the codebase consists of *beyond* the UI: areas,
    the API surface, the data model, jobs, integrations. Regenerated by the
    `codebase-map` skill.
  </Card>
  <Card title="App sitemap" icon="map">
    **`/synclair/pages`** — every view/route the app serves, with a live preview of
    each, the navigation edges, and the components each one composes. Regenerated by
    the `pages-map` skill.
  </Card>
  <Card title="Hygiene" icon="broom">
    **`/synclair/hygiene`** — where host code steps outside its own foundation
    (inline styles, raw colors, bypassed primitives). An advisory sweep.
  </Card>
  <Card title="Library (surfaces)" icon="layer-group">
    **`/synclair/library`** — for multi-frontend products (e.g. web + React Native
    sharing a backend), a per-surface drill-in. Invisible for single-frontend projects.
  </Card>
</CardGroup>

## Operations

<CardGroup cols={2}>
  <Card title="AI setup" icon="robot">
    **`/synclair/ai-setup`** — every skill and agent available, classified by
    category and origin, so a human can see what each is and where it came from.
  </Card>
  <Card title="GitHub" icon="github">
    **`/synclair/github`** — recent commits and diffs. Git is the shared log; this
    just reads it.
  </Card>
  <Card title="Environment" icon="server">
    **`/synclair/environment`** — the running stack, plus the opt-in "call home"
    check for whether your clone is behind the upstream foundation.
  </Card>
  <Card title="⌘K search" icon="magnifying-glass">
    A command palette across everything in the hub — components, tokens, docs, pages.
  </Card>
</CardGroup>

## For existing-app (watcher) mode

When Synclair runs beside a host app, a few sections take on a host flavor:

- **Components** shows host components tagged **"Host"** — documented from the host's
  source (props derived live) without importing its code.
- **Hygiene** and the **host catalog** freshness are checked against the host repo,
  so the catalog can't quietly go stale.
